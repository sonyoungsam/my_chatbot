# 🤖 My First Chat Bot

로컬 네트워크에 구축된 **자체 호스팅 LLM**(Qwen3)과 대화할 수 있는 [Streamlit](https://streamlit.io/) 기반 챗봇입니다.
OpenAI 호환 API를 사용하므로, 엔드포인트만 바꾸면 다른 로컬/사설 LLM 서버에도 쉽게 연결할 수 있습니다.

단순 챗봇을 넘어, **실시간 웹 검색·날씨·주식 시세·특정 회사 전용 RAG**까지 붙어 있어 로컬 LLM 혼자서는 알 수 없는 정보도 답변에 반영합니다.

## ✨ 주요 기능

| 기능 | 설명 |
|---|---|
| 💬 실시간 스트리밍 응답 | 토큰이 생성되는 즉시 화면에 순차 출력 (`st.write_stream` 기반, DOM 오류 방지용 업데이트 스로틀링 포함) |
| 🧠 Reasoning(생각 과정) 자동 숨김 | Qwen3의 `<think>...</think>` 블록을 요청 단계 + 스트리밍 필터 이중으로 차단, 최종 답변만 표시 |
| 🗂️ 다중 대화(멀티 세션) | ChatGPT처럼 여러 대화를 만들고 사이드바 목록에서 전환. 첫 메시지로 제목 자동 생성 |
| 🔎 실시간 웹 검색 | 네이버 검색(비공식, AI 요약/기업정보 추출)을 우선 사용하고, 실패 시 DuckDuckGo(뉴스+텍스트, 재시도 포함)로 대체 |
| 🌤️ 실시간 날씨 | Nominatim 지오코딩 + Open-Meteo API. 현재 날씨, 12시간 시간대별 예보, 14일 일별 예보, 한국 행정구역명 인식 |
| 📈 실시간 주식 시세 | Yahoo Finance(`yfinance`). 국내(코스피/코스닥)·해외 종목, 한글 별칭/종목코드/티커 모두 인식, 네이버 검색 폴백, 기간별 추이 차트 |
| 🏢 크리스피드 전용 RAG | crespeed.com을 크롤링해 임베딩 인덱스를 만들고, 관련 질문에 최우선으로 근거 있는 답변 제공 |
| 🎛️ 사이드바 LLM 옵션 | Temperature, Top P, Max output tokens, System prompt를 UI에서 즉시 조절 |
| ⚠️ 정직성 안전장치 | 검색/API가 실패하면 화면에 경고를 직접 표시해, 모델이 "검색 결과를 바탕으로"라며 지어내는 것을 방지 |

```
│  (채팅 생성)      │ │ 서버(RAG용)  │ │ Open-Meteo    │ │ + 네이버 검색   │ │ DuckDuckGo      │
│  Qwen3.6-35B-FP8 │ │ bge-m3-ko   │ │ (날씨)         │ │ (주식)          │ │ (일반 웹검색)     │
│ :18000/v1        │ │ :18001/v1   │ │               │ │                │ │                 │
└───────────────┘ └─────────────┘ └──────────────┘ └───────────────┘ └────────────────┘
```

- **Frontend / Orchestration**: Streamlit (`app.py`) — 채팅 UI, 세션 상태, 스트리밍 렌더링, 각 기능의 우선순위 라우팅

- **임베딩 LLM**: 별도 OpenAI 호환 서버(`dragonkue/bge-m3-ko`) — 크리스피드 RAG 검색용
- **외부 데이터 소스**: 모두 무료·API 키 불필요 (Nominatim, Open-Meteo, yfinance, 네이버 비공식 검색, DuckDuckGo)

## 🚀 시작하기

### 1. 사전 요구사항

- Python 3.10 이상
- 사설망에서 접근 가능한 OpenAI 호환 LLM 서버 (`http://192.168.0.201:18000/v1`)
- (크리스피드 RAG 기능용) 임베딩 서버 (`http://192.168.0.201:18001/v1`, `dragonkue/bge-m3-ko`) — 없어도 다른 기능은 정상 동작

### 2. 가상환경 생성 및 활성화

**Windows (PowerShell)**
```powershell
cd d:\projects\my_chatbot
python -m venv venv
venv\Scripts\activate
```

**macOS / Linux**
```bash
cd my_chatbot
python3 -m venv venv
source venv/bin/activate
```

### 3. 의존성 설치

```bash
pip install -r requirements.txt
```

### 4. 앱 실행

```bash
streamlit run app.py
```

실행 후 터미널에 표시되는 주소로 접속하면 됩니다.

```
Local URL: http://localhost:8501
```

브라우저가 자동으로 열리지 않으면 위 주소를 직접 입력해서 접속하세요.

### 5. (Windows) 아이콘 더블클릭으로 실행하기

매번 터미널을 열지 않아도 되도록 실행 파일과 바탕화면 아이콘을 만들어 두었습니다.

- [run_chatbot.bat](run_chatbot.bat) — 더블클릭하면 가상환경 활성화 후 `streamlit run app.py`를 자동 실행합니다. (최초 1회는 `venv`가 준비되어 있어야 합니다.)
- 바탕화면의 **"My Chat Bot"** 바로가기 — [assets/icon.ico](assets/icon.ico) 아이콘을 사용하는 `run_chatbot.bat` 바로가기입니다.

다른 PC에 새로 바로가기를 만들고 싶다면 PowerShell에서:

```powershell
$WshShell = New-Object -ComObject WScript.Shell
$desktop = [Environment]::GetFolderPath("Desktop")
$Shortcut = $WshShell.CreateShortcut("$desktop\My Chat Bot.lnk")
$Shortcut.TargetPath = "<프로젝트 경로>\run_chatbot.bat"
$Shortcut.WorkingDirectory = "<프로젝트 경로>"
$Shortcut.IconLocation = "<프로젝트 경로>\assets\icon.ico"
$Shortcut.Save()
```

[app.py](app.py) 상단의 상수를 수정하면 다른 서버·모델로 전환할 수 있습니다.

```python
EMBEDDING_BASE_URL = "http://192.168.0.201:18001/v1"
```

### 사이드바 옵션

**LLM 옵션**

| 옵션 | 범위 | 기본값 | 설명 |
|---|---|---|---|
| Temperature | 0.0 ~ 2.0 | 0.3 | 값이 높을수록 응답이 창의적/무작위적 |
| Top P | 0.0 ~ 1.0 | 0.9 | 누적 확률 기반 샘플링 범위 |
| Max output tokens | 64 ~ 8192 | 1024 | 응답 최대 길이 |
| System prompt | 자유 입력 | (하단 참고) | 모델의 역할/성격 지정. "모르면 모른다고 답하라"는 지침이 기본 포함 |

**🔎 실시간 정보**

| 옵션 | 기본값 | 설명 |
|---|---|---|
| 웹 검색으로 답변 보강 | ON | 질문 내용으로 웹을 검색해 최신 정보를 답변에 반영 |
| 🌤️ 날씨 질문에 실시간 날씨 API 사용 | ON | '~날씨' 포함 질문에 Nominatim+Open-Meteo로 정확한 값 조회 |
| 기본 날씨 지역 | `Seoul` | 질문에 지역명이 없을 때 사용할 기본 지역 |
| 📈 주식 질문에 실시간 시세 API 사용 | ON | '~주가/시세' 포함 질문에 Yahoo Finance로 정확한 값 조회 |
| 🏢 크리스피드 질문에 공식 사이트 RAG 사용 | ON | '크리스피드' 포함 질문에 crespeed.com 임베딩 검색 결과 최우선 사용 |
| 🔄 크리스피드 사이트 다시 수집 | 버튼 | 캐시를 지우고 다음 질문부터 사이트를 새로 크롤링/임베딩 |

- **🆕 새 대화**: 현재 대화는 목록에 남긴 채 새 대화 시작
- **대화 목록**: 과거 대화를 클릭해 다시 열람/이어가기

## 🧠 주요 기능 상세

### 1. Reasoning(생각 과정) 숨김 처리

Qwen3 계열 모델은 최종 답변 전에 `<think>...</think>` 형태로 내부 추론 과정을 함께 생성하는 경우가 있습니다. 두 단계로 걸러내어 **최종 답변만** 보여줍니다.

1. **요청 시 비활성화**: API 호출에 `extra_body={"chat_template_kwargs": {"enable_thinking": False}}`를 포함해, 서버가 지원하면 애초에 생각 과정을 생성하지 않도록 요청
2. **스트리밍 필터 (안전장치)**: 그래도 `<think>...</think>` 블록이 오면 스트리밍 도중 실시간으로 감지해 제거 (토큰 경계에서 태그가 잘리는 경우까지 처리)

대화 히스토리에도 최종 답변만 저장되어, 다음 턴에 이전 생각 과정이 재전송되지 않습니다.

### 2. 실시간 웹 검색 (네이버 우선 + DuckDuckGo 대체)

- 사용자 문장에서 "~에 대해 설명해줘", "~는 뭐야" 같은 지시문을 잘라내고 핵심 키워드만 검색어로 사용 (검색엔진은 자연어 질문이 아니라 키워드에 반응)
- **네이버 통합검색** 결과 페이지에 내장된 JSON에서 AI 요약·기업정보(업종/사원수/대표자명 등)를 정규식으로 추출 (비공식, API 키 불필요)
- 네이버에서 못 찾으면 **DuckDuckGo**(뉴스+텍스트 결과 병합, 최대 5회 재시도)로 대체
- 검색이 끝내 실패하면, 모델이 "검색 결과를 바탕으로"라며 지어내지 못하도록 **화면에 직접 경고 배너**를 표시

### 3. 실시간 날씨

- **Nominatim(OpenStreetMap)**으로 지역명 → 좌표 변환 (Open-Meteo 자체 지오코딩은 한국 "구" 단위를 인식 못하거나 동명 지역을 다른 나라로 잘못 매칭하는 문제가 있어 교체)
- 문장에서 조사를 뗀 뒤, 시/도/구/군/읍/면/동 같은 행정구역 접미사가 붙은 단어를 최우선 후보로 삼고, 실제 지오코딩 성공 여부로 검증
- **Open-Meteo**로 현재 날씨 + 12시간 시간대별 예보 + 14일 일별 예보 조회
- 예: "송파구 날씨 알려줘", "강원도 홍천군 주말 날씨", "부산 1주일 예보"

### 4. 실시간 주식 시세

- 국내 종목은 한글 별칭 사전(삼성전자, 하이닉스 등 약 60개) + 6자리 종목코드 + 네이버 금융 자동완성 검색(사전에 없는 종목 폴백)으로 티커 인식
- 해외 종목은 영문 티커(`AAPL`)를 직접 인식하거나 네이버 검색으로 매핑
- **Yahoo Finance**(`yfinance`)로 현재가/전일종가/등락률/고가·저가/거래량/시가총액 조회, `quote_type == "EQUITY"`만 채택해 ETF/펀드 오매칭 방지
- "추이/차트/1달간/일주일/올해" 같은 기간 표현이 있으면 기간별 시세 추이와 꺾은선 차트도 함께 표시
- 실시간이 아니라 **최대 15~20분 지연**된 데이터임을 답변에 명시하도록 지침 포함

### 5. 크리스피드(CRESPEED) 전용 RAG

특정 회사(crespeed.com)에 대한 질문에는 일반 웹 검색보다 회사 공식 사이트 원문을 우선 사용합니다.

1. **크롤링**: crespeed.com 내부를 얕게 크롤링 (`<a href>` 링크뿐 아니라 `<li onclick="javascript:fnGnbLink(...)">` 형태의 자바스크립트 메뉴까지 인식)
2. **정제**: 여러 페이지에 반복되는 메뉴/푸터 텍스트를 빈도 기반으로 자동 감지해 제거하고, 남은 본문을 청크로 분할
3. **임베딩 인덱스**: 청크를 `dragonkue/bge-m3-ko` 임베딩 서버로 벡터화하고, 질문과의 코사인 유사도로 상위 6개를 검색
4. **캐싱**: 크롤링+임베딩 결과를 로컬 파일(`.crespeed_cache.json`, 7일 캐시)에 저장해 매번 다시 수집하지 않음. 사이드바의 "🔄 다시 수집" 버튼으로 강제 새로고침 가능
5. 다른 회사를 대상으로 재사용하려면 [app.py](app.py)의 `CRESPEED_BASE_URL`, `CRESPEED_DOMAIN`, `CRESPEED_KEYWORDS`를 바꾸면 됩니다.

## 🩺 문제 해결 (Troubleshooting)

| 증상 | 원인 / 해결 |
|---|---|
| `오류가 발생했습니다: Connection error.` | LLM 서버(`192.168.0.201:18000`)가 꺼져 있거나 같은 네트워크(사설망)에 있지 않은 경우입니다. 서버 상태와 IP/포트를 확인하세요. |
| 첫 실행 시 터미널이 이메일 입력에서 멈춤 | Streamlit 최초 실행 시 나오는 온보딩 프롬프트입니다. `streamlit run app.py --browser.gatherUsageStats false`로 실행하거나 빈 값으로 Enter를 누르면 됩니다. |
| 생각 과정(`<think>`)이 그대로 보임 | 서버가 `enable_thinking` 옵션을 지원하지 않을 수 있습니다. 스트리밍 필터가 안전장치로 동작하지만, 서버가 다른 태그/필드를 쓰는 경우 [app.py](app.py)의 필터 로직 조정이 필요합니다. |
| 응답이 중간에 잘림 | 사이드바의 `Max output tokens` 값을 늘려보세요. |
| 포트 충돌 (`8501` 사용 중) | `streamlit run app.py --server.port 8502`처럼 다른 포트를 지정하세요. |
| "⚠️ 실시간 웹 검색에 실패했습니다" 경고가 자주 뜸 | 네이버/DuckDuckGo 모두 API 키 없는 비공식 검색이라 가끔 응답이 느리거나 실패할 수 있습니다. 잠시 후 다시 물어보세요. |
| 날씨/주식 질문인데 엉뚱한 지역·종목으로 답함 | 문장에서 핵심 키워드 추출이 실패한 경우입니다. 지역명·종목명을 더 명확히 (예: "그 회사" 대신 실제 이름) 적어보세요. |
